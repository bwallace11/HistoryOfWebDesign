What Was Happening

In 1994, the World Wide Web Consortium (W3C) formed to guide web standards. At the time, HTML carried structure and a lot of ugly presentation because authors wanted control and browsers were competing for attention.

In that environment, Håkon Wium Lie proposed Cascading Style Sheets (CSS) as a way to separate document structure (HTML) from presentation (style rules). Bert Bos quickly became a major collaborator, and CSS work moved through W3C.

Major People and Groups

  • Håkon Wium Lie (W3C / later Opera): proposed CSS (1994)
  • Bert Bos (W3C): co-author/editor in early CSS work
  • Tim Berners-Lee (W3C): supported architecture separating structure and presentation
  • W3C: organized CSS work toward Recommendation status

Why CSS Was Needed

  1. HTML was getting stuffed with presentation: <font>, <center>, color attributes.
  2. Sites needed consistent visual rules across many pages (not manual formatting everywhere).
  3. Users sometimes needed control (fonts, contrast) for accessibility and preference.
  4. Print vs screen required different presentation rules.

What Authors Actually Did

  • Presentational HTML everywhere: <font>, <center>, bgcolor, align.
  • Tables as layout scaffolding (even before it was “officially” encouraged).
  • Spacer GIFs to force spacing because layout control was weak.
  • “Best viewed in…” badges because consistent rendering was a fantasy.

Early CSS Ideas (Tiny Example)

Browsers had little consistent support, so even simple things could be unreliable: 

body { font-family: serif; }
h1 { font-size: 20pt; }
