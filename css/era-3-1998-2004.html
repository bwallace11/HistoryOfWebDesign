1998–2004: Browser wars and “why won’t this look the same?”

This era is where CSS exists, but browser support is uneven. Developers often fell back to HTML layout tricks because CSS rendering differed across browsers.

Key theme: CSS is real, but implementation is a mess, so author practices become a mix of CSS and HTML workarounds.

Important things to note

  • CSS Level 2 (1998) exists, but browsers don’t implement everything consistently. :contentReference[oaicite:10]{index=10}
  • Developers used “fallback thinking”: if CSS breaks, the page must still be readable.
  • Standards advocacy grows: people start pushing browsers to follow specs more closely.

Major players

  • W3C CSS Working Group forming CSS direction and errata work. :contentReference[oaicite:11]{index=11}
  • Håkon Wium Lie, Bert Bos, and other W3C editors/staff working on CSS. :contentReference[oaicite:12]{index=12}
  • Browser teams (Microsoft, Netscape/Mozilla, Opera) making their own choices and bugs.

Era CSS rule: No modern CSS here. Only basic typography, colors, and simple boxes.
