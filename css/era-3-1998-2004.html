1998–2001: CSS Level 2 (CSS2) and the Browser Wars

Key Event

On May 12, 1998, the W3C published Cascading Style Sheets, level 2 (CSS2) as a Recommendation. CSS2 expanded CSS from styling text into controlling layout behavior.

Spec Editors and Groups

  • Bert Bos – CSS2 editor
  • Håkon Wium Lie – CSS2 editor
  • Chris Lilley – CSS2 editor
  • Ian Jacobs – CSS2 editor
  • W3C CSS Working Group

Browser/Engine Teams (Why CSS2 Hurt)

Trident (Microsoft Internet Explorer)
Strong market power, uneven standards behavior, lots of quirks.
Gecko (Mozilla)
Rebuilt to replace old Netscape engine; big push toward standards.
Presto (Opera)
Often strong standards support, smaller market share.

What CSS2 Added (Big Deals)

  1. Positioning: absolute, relative, fixed
  2. Z-index layering
  3. Media types (print vs screen)
  4. More precise box model definitions
  5. Aural styles (ambitious, barely implemented)

What Authors Actually Did (Hacks + Survival)

  • Tables for layout stayed dominant.
  • Float layouts started appearing, but were fragile.
  • Internet Explorer box model hacks (targeting broken width calculations).
  • “Quirks mode” behavior appeared based on DOCTYPE presence.
  • Proprietary features: Microsoft filters, nonstandard behaviors, vendor differences.

Era CSS2 Example (Careful)

#nav { position: relative; }
.note { border: 1px solid black; padding: 8px; }
