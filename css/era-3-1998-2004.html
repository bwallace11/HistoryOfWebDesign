Era 3: CSS Level 2 and the browser wars (1998-2001)

CSS Level 2 becomes a W3C recommendation on 12 May 1998.
Main Hub CSS Home | 1994-1996 | 1996-1998 | 1998-2004 | 2004-2011 | 2011-2017 | 2017-2026 | Glossary | Bibliography | Contributors | Discussion

What CSS2 tried to deliver

  • Positioning (absolute, relative, fixed)
  • Layering with z-index
  • Media types (print vs screen)
  • More selectors and richer formatting

Reality check

CSS2 was ambitious. Browser support was inconsistent, and authors lived in workaround land.

Major players (spec and institutions)

CSS2 is edited by Bert Bos, Håkon Wium Lie, Chris Lilley, and Ian Jacobs. The World Wide Web Consortium (W3C) coordinates the recommendation track.

Browser wars impact

Competing browsers shipped overlapping CSS features with different bugs and different interpretations. Authors often built two styles: one for a dominant browser, one for "everything else."

Big lesson: the spec existing did not mean the web behaved the same everywhere.

What authors actually did (1998-2001)

  • Table layout still ruled: CSS layout was not dependable, so tables stayed the default layout tool.
  • Early float layouts: floated columns existed but required careful clearing.
  • Browser sniffing: different style sheets or conditional hacks for different engines.
  • Print styles: experimented with media types, but support varied.

Why this era matters

CSS2 set the conceptual foundation for modern layout and media styling, even though implementation lagged. It also showed that interoperability requires tests, shared behavior, and patience.

CSS2 Recommendation date: 12 May 1998.
