2001–2011: CSS Level 2 Revision 1 (CSS 2.1) and the Long Cleanup

This era is dominated by a painful truth: CSS existed, but cross-browser consistency did not. The goal of Cascading Style Sheets Level 2 Revision 1 (CSS 2.1) was to document what browsers actually implemented and remove or clarify features that were broken or never shipped.

Key Event

In June 2011, CSS 2.1 became a W3C Recommendation, solidifying the baseline.

Who Shaped This Era

  • W3C CSS Working Group – maintained CSS 2.1 work for years
  • Browser implementers – forced reality checks through actual engine behavior
  • Standards advocates – pushed “write standards, not browser hacks” culture

Engine Teams (What Controlled Reality)

Trident (Internet Explorer 6–8)
Dominant in market share, source of many compatibility constraints.
Gecko (Firefox)
Improved standards behavior and developer tools helped shift culture.
WebKit (Safari and early Chrome)
Fast iteration, later forked into Blink (2013, next era).
Presto (Opera)
Consistently strong standards support, smaller market footprint.

What Authors Actually Did (The Greatest Hits of Pain)

  • Float layouts + clearfix hacks to keep containers from collapsing.
  • HasLayout workarounds in Internet Explorer (triggering layout modes).
  • Conditional comments to serve CSS only to Internet Explorer.
  • Star HTML hack and underscore hack to target IE parsing behavior.
  • PNG transparency hacks and filters for old IE.
  • Reset styles to normalize default browser styling.

CSS 2.1-Era Example

.float-left { float: left; width: 200px; }
.clearfix:after { content: "."; display: block; height: 0; clear: both; visibility: hidden; }

Why This Era Matters

CSS 2.1 is the reason modern CSS could move forward without collapsing under incompatible interpretations. It basically turned chaos into a shared baseline.
