Era 4: CSS 2.1 baseline and the fixed-width centered web (2004-2011)

CSS Level 2 Revision 1 (CSS 2.1) becomes a W3C recommendation on 7 June 2011, but it defines the baseline behavior that authors and browsers stabilized during the 2000s.

Era style pattern

  • 960px centered wrapper
  • Float columns
  • Clearfix hacks
  • "Header / Nav / Content / Footer" layout

The "margin center" hack

width: 960px;
margin: 0 auto;

Why this era happened

After the late-1990s chaos, authors wanted a dependable baseline. CSS features that were too buggy or too inconsistently implemented got avoided. The community converged on "safe" CSS: floats, clear fixes, predictable typography, and fixed-width centered layouts.

Major players (spec editors and community)

  • Bert Bos: long-term CSS editor and W3C contributor.
  • Tantek Çelik and Ian Hickson: listed as editors on CSS 2.1 spec.
  • Web Standards Project (WaSP): advocacy pushing browser support and standards compliance.
  • Browser engine teams: Trident (Microsoft), Gecko (Mozilla), WebKit (Apple), Presto (Opera).

What authors actually did (2001-2010)

  • 960px grid layouts: fixed containers, often 960px, centered with margin: 0 auto.
  • Float columns + clearfix: floats for layout, clearfix hacks to contain them.
  • Image-based rounded corners: before border-radius was reliable, corners were sliced images.
  • "HasLayout" and IE fixes: special hacks for older Microsoft Internet Explorer behavior.
  • Reset CSS: normalize browser default margins/paddings to reduce surprises.

Why CSS 2.1 matters

CSS 2.1 is a cleanup and stabilization step: it clarified behavior and removed or de-emphasized features that never worked consistently. It became the "boring baseline" that made modern CSS possible.

Artifact: The 960 Grid System (2008)

Title: 960 Grid System
Year: 2008
Creator: Nathan Smith

The 960 Grid System was a CSS framework that gave designers a standardized 12- or 16-column grid within a 960px-wide container. It became the dominant layout approach of this era because 960 divides evenly by 2, 3, 4, 5, 6, 8, 10, 12, 15, and 16.

Example pattern:

.container_12 { width: 960px; margin: 0 auto; }
.grid_6 { float: left; width: 460px; margin: 0 10px; }

Why it mattered: Grid systems brought order to the chaos of float-based layout. They established conventions for column widths, gutters, and alignment that influenced every CSS framework that followed, including Bootstrap.

Artifact: The Clearfix Hack

Title: "Micro clearfix hack"
Year: 2004-2011
Notable version by: Nicolas Gallagher (2011)

When float-based columns were used for layout, the parent container would collapse to zero height because floated elements are removed from normal flow. The "clearfix" hack used pseudoelements or overflow tricks to force the parent to contain its floated children.

Why it mattered: The clearfix hack is a symbol of this era's CSS limitations. Developers had to fight the language to achieve basic layouts. This frustration drove demand for Flexbox and Grid.

Limitations designers faced

  • No native rounded corners; required sliced corner images.
  • No shadows without images or hacks.
  • No gradients; background images for every subtle gradient.
  • Equal-height columns required JavaScript or "faux columns" background tricks.
  • IE6 refused to die, requiring special hacks for years after newer browsers appeared.
Key reference: CSS 2.1 W3C recommendation date: 7 June 2011.
