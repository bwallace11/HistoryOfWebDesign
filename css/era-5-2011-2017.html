Era 5: CSS3 modules, responsive design, modern UI patterns (2011-2017)

CSS becomes modular: instead of "one CSS3," you get stable modules like backgrounds, borders, selectors, media queries, and transforms.

What changes

  • Responsive design becomes mainstream (media queries + flexible layouts).
  • Rounded corners and shadows become normal: border-radius, box-shadow.
  • Visual polish without images: gradients, transforms, transitions.
  • Flexbox arrives for one-dimensional layout.

Major contributors

  • Eric Meyer: author of key CSS books and creator of the "Meyer reset," helping popularize practical, standards-based CSS.
  • Rachel Andrew: CSS Grid educator and standards communicator.
  • Jen Simmons: major advocate/teacher for modern layout at Mozilla.
  • Lea Verou: CSS educator, tooling, and standards work.
  • Elika J. Etemad (fantasai): major CSS editor and spec leader.
  • Ethan Marcotte: coined "responsive web design."

What authors actually did

  • Mobile-first layouts: design for small screens, then add rules for larger screens.
  • "Flat design": minimal decoration, bold type, strong spacing.
  • Frameworks: prebuilt grids and UI patterns become common.
  • Vendor prefixes: write the same property several times (example: -webkit-).

Artifact: "Responsive Web Design" Article (2010)

Title: "Responsive Web Design"
Year: 2010
Author: Ethan Marcotte, A List Apart

Marcotte proposed three ingredients: fluid grids (percentage widths), flexible images (max-width: 100%), and CSS3 media queries. Together, these techniques let a single codebase adapt to phones, tablets, and desktops.

Example media query from this era:

@media (max-width: 600px) {
  .sidebar { display: none; }
  .content { width: 100%; }
}

Why it mattered: Before responsive design, companies built separate mobile sites (m.example.com). Marcotte showed that one site could serve all devices, which became the industry standard.

Artifact: Bootstrap CSS Framework (2011)

Title: Bootstrap
Year: 2011
Creators: Mark Otto and Jacob Thornton at Twitter

Bootstrap was an internal tool at Twitter that became the most popular CSS framework in the world. It provided a responsive 12-column grid, pre-styled buttons, forms, navigation, and modals. Developers could build professional-looking sites without writing much custom CSS.

Why it mattered: Bootstrap democratized responsive web design. It also showed the demand for layout tools that CSS itself did not yet provide. The popularity of frameworks pushed the CSS Working Group to prioritize native layout features like Flexbox and Grid.

Why this era replaced the previous one

The fixed-width 960px layouts of the CSS 2.1 era could not handle the explosion of mobile devices. When the iPhone launched in 2007, most websites were unreadable on phone screens. CSS3 media queries and Flexbox gave authors the tools to build layouts that adapted to any screen size.

Limitations designers still faced

  • Flexbox was one-dimensional only (rows OR columns, not both).
  • Vendor prefixes created maintenance headaches.
  • Old IE versions still required fallbacks for years.
  • Complex grid layouts still required frameworks or hacks.
