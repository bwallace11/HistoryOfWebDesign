2017–2026: Grid, modern selectors, and the “you can finally do it in CSS” era

CSS Grid becomes a real standard-era layout system around 2017, with the W3C process moving it through key stages and browsers shipping broad support soon after. :contentReference[oaicite:23]{index=23}

What Grid changed

  • Two-dimensional layout (rows and columns) without table hacks.
  • Cleaner page structure: fewer wrapper divs just for layout.
  • Real alignment tools, not “please line up, I’m begging you.”

Big modern features (2017–2026)

  • CSS Custom Properties (“CSS variables”) become normal tooling in real projects. :contentReference[oaicite:24]{index=24}
  • Container query work matures through CSSWG drafts and modules. :contentReference[oaicite:25]{index=25}
  • Selectors keep evolving through levels (the spec keeps moving because it’s updated often). :contentReference[oaicite:26]{index=26}

Major players

  • W3C CSS Working Group and its participants. :contentReference[oaicite:27]{index=27}
  • Editors and implementers from browser teams (Google, Apple, Mozilla, Microsoft, Igalia, etc.). :contentReference[oaicite:28]{index=28}

Important reality check

Modern CSS is not “one version.” It’s a living pile of modules. The W3C publishes snapshots, drafts, and recommendations, and browsers ship features gradually. :contentReference[oaicite:29]{index=29}
