HTML 2.0, 3.0, and 3.2

Standardizing early web pages and the first browser wars

The mid-1990s transformed the web from a research tool into a commercial medium. Netscape and Microsoft raced to add features, often inventing proprietary tags that only worked in their browser. The W3C tried to keep up by publishing standards, but browsers moved faster than committees. This era gave us tables, images everywhere, and the infamous <blink> and <marquee> tags.

Version Years Highlights
HTML 2.0 1995 First formal HTML standard from the IETF; defined core tags for forms, images, and basic document structure.
HTML 3.0 (draft) 1995-1996 Ambitious draft led by Dave Raggett, adding tables, math, and more; abandoned because it was too large and browsers could not keep up.
HTML 3.2 1997 First W3C HTML recommendation, code-named "WILBUR"; standardized many features in use, including tables, images, and some presentational elements.

Artifact: Space Jam Official Website (1996)

Title: Space Jam Movie Website
Year: 1996
Creator: Warner Bros.

The official website for the movie Space Jam is one of the most famous surviving examples of 1990s web design. It used table-based layout, image maps, animated GIFs, and bright colors that defined the era. The site was never redesigned and remained online for over 25 years in its original form.

How it was built: The entire layout was a table. Navigation was an image map (a single image with clickable regions). Every visual element was a GIF image. There was no CSS.

Example of table-based layout from this era:

<table>
  <tr>
    <td><img src="logo.gif"></td>
    <td>Navigation</td>
  </tr>
</table>

Why it mattered: Space Jam showed that the web was becoming a marketing platform, not just a research tool. Its survival as an unmodified artifact makes it a time capsule of 1990s web techniques.


Artifact: GeoCities Personal Pages (1995-1999)

Title: GeoCities
Year: 1995 (launched)
Creator: Beverly Hills Internet (later Yahoo!)

GeoCities gave millions of people their first personal web page. Users picked a "neighborhood" (like Hollywood, SiliconValley, or Area51) and built pages using the HTML they knew: tables for layout, <font> tags for colors, animated GIF collections, visitor counters, and guestbooks.

GeoCities pages were messy, colorful, and deeply personal. They represented the web at its most democratic: anyone could publish, and the tools were simple enough for amateurs to use.

Why it mattered: GeoCities proved that ordinary people wanted to create, not just consume, on the web. When Yahoo! shut it down in 2009, archivists scrambled to preserve millions of pages as cultural artifacts.


Why this era replaced the previous one

Early HTML (1990-1994) could only create plain text documents with links. Authors had no control over how pages looked. As the web went commercial, businesses and individuals demanded visual control. Browsers responded by adding tags like <font>, <center>, and <table>. These features were not elegant, but they were the only tools available before CSS.

Limitations designers faced

  • Tables were the only way to create multi-column layouts.
  • Every visual change required HTML tags mixed into the content.
  • Changing the look of a site meant editing every single page by hand.
  • Different browsers rendered the same HTML differently.
  • Accessibility was not considered; screen readers struggled with table layouts.

Major contributors

  • Dave Raggett - key editor of early HTML specifications.
  • Lou Montulli - creator of the Lynx text browser and early Netscape engineer.
  • Marc Andreessen - co-founded Netscape; pushed for the <img> tag and visual web features.
  • Netscape Communications - introduced many proprietary tags during the browser wars.
  • Microsoft - Internet Explorer team competed with Netscape.
  • W3C (World Wide Web Consortium) - founded 1994 to coordinate standards.

Important changes in this era

  • Introduction of tables that authors quickly used for page layout as well as data.
  • Widespread use of presentational elements such as font, center.
  • Growing tension between browser-specific tags and the need for common standards.
  • Images became central to web design with <img>, image maps, and animated GIFs.
