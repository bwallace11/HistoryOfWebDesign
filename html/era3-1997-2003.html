HTML 4.0, HTML 4.01, and XHTML 1.x

The push toward structure, accessibility, and separation of concerns

HTML 4.0 (1997) and HTML 4.01 (1999) marked the transition from purely presentational markup to more semantic, internationalized, and accessible HTML. The same element set was later re-expressed as XHTML 1.0 and XHTML 1.1 using XML syntax.

This was the era when the web standards community began arguing that HTML should describe what content is, not how it looks. CSS was supposed to handle appearance. In practice, most sites still mixed structure and presentation heavily.

Why this era replaced the previous one

HTML 3.2 had standardized presentational tags like font and center, but maintaining large sites built this way was painful. Changing a font across 500 pages meant editing 500 files. HTML 4.0 introduced the idea that styling should be separated from markup and handled by CSS. It deprecated many presentational elements and encouraged authors to use class and id attributes with external stylesheets.

Artifact: Web Standards Movement and the Browser Wars

Title: The Web Standards Project (WaSP)
Year: 1998
Founders: George Olsen, Glenn Davis, Jeffrey Zeldman

The Web Standards Project was a grassroots advocacy group that pressured browser makers to implement W3C standards consistently. Before WaSP, web developers had to build separate versions of every site for Netscape and Internet Explorer because each browser interpreted HTML and CSS differently.

Why it mattered: WaSP helped establish the principle that the web should be built on open, interoperable standards rather than proprietary browser features. Their advocacy directly influenced browser makers to improve standards compliance.


Artifact: The Frames Experiment

Title: Frameset-based website layouts
Year: 1997-2003
Technology: HTML frames

HTML 4.0 formalized frames, which split the browser window into multiple independent panes. A typical frameset site had a navigation frame on the left and a content frame on the right. Clicking a link in the navigation frame loaded new content in the content frame without refreshing the entire page.

Example frameset code:

<frameset cols="200,*">
  <frame src="nav.html" name="nav">
  <frame src="main.html" name="content">
  <noframes>
    <p>Your browser does not support frames.</p>
  </noframes>
</frameset>

Why it died: Frames broke bookmarking (the URL never changed), confused search engines, and made accessibility nearly impossible. They were deprecated in HTML5.


Limitations designers faced

  • CSS support was inconsistent across browsers, so many authors still used table layouts.
  • Internet Explorer and Netscape rendered the same page differently.
  • XHTML required strict XML syntax, but served as text/html it was still parsed as tag soup.
  • Frames seemed clever but broke fundamental web conventions like bookmarks and the back button.
  • Accessibility was discussed in specs but rarely implemented in practice.

Goals of HTML 4.x

  • Separate structure from presentation and encourage the use of CSS for styling.
  • Support internationalization, accessibility, and device independence through attributes like lang and better form semantics.
  • Clean up many browser-specific tags from the 3.2 era and mark them as deprecated.

Major contributors

  • W3C HTML Working Group - developed HTML 4.0/4.01 and XHTML 1.0/1.1.
  • Tim Berners-Lee - W3C Director, guiding the standards direction for HTML.
  • Dave Raggett and other editors - authors of W3C HTML specifications and tutorials.
  • Jeffrey Zeldman and the Web Standards Project - advocated for consistent browser behavior.
  • Accessibility and internationalization groups within W3C - pushed for better markup for screen readers and non-English text.

Important elements and concepts

  • div and span for generic block and inline grouping with class and id attributes.
  • Expanded forms: label, fieldset, legend, optgroup for structured, accessible forms.
  • Frames and inline frames: frame, frameset, noframes, iframe.
  • Semantic inline elements like abbr, acronym, cite, q, and dfn.
