XHTML 2.0 vs the emergence of HTML5

In the early 2000s, the W3C focused on XHTML 2.0, a proposed successor to XHTML 1.x that broke compatibility with existing web content. At the same time, browser vendors grew frustrated that the official road map did not match real-world HTML usage.

The XHTML 2.0 path

  • Removed familiar elements like img in favor of new XML-based patterns.
  • Required stricter XML rules, which many existing sites did not follow.
  • Ultimately abandoned; XHTML 2.0 never shipped in mainstream browsers.

The birth of WHATWG HTML5

  • In 2004, engineers from Apple, Mozilla, and Opera created the WHATWG (Web Hypertext Application Technology Working Group).
  • They started work on "Web Applications 1.0," which evolved into HTML5, focusing on backward compatibility and application features.
  • The W3C later restarted its HTML Working Group and collaborated with WHATWG on HTML5 drafts.

Major contributors

  • Ian Hickson - principal editor of the HTML5 specification in WHATWG and W3C drafts.
  • Browser vendor representatives from Apple, Mozilla, Opera, Microsoft, and later Google - pushed for a practical, compatible HTML standard.
