Glossary of HTML Terms

Hypertext

Text that contains links to other documents. The ability to click a link and navigate to another document is the core idea that motivated the creation of HTML and the World Wide Web.

Markup

Instructions embedded within a document that describe how text should be structured or displayed. HTML uses angle-bracket tags as its markup syntax.

Element

A single HTML construct such as a heading, paragraph, or link. Most elements have an opening tag, content, and a closing tag: <p>text</p>.

Attribute

Additional information placed inside an opening tag to configure an element, such as href in <a href="...">.

DTD (Document Type Definition)

A formal grammar that specifies the valid elements and structure of an HTML document. HTML 4.01 and XHTML used DTDs extensively; HTML5 replaced them with a browser-compatible parsing algorithm.

SGML (Standard Generalized Markup Language)

The international standard for describing markup languages. HTML was originally defined as an SGML application. XHTML moved to XML; HTML5 introduced its own independent parsing rules.

W3C (World Wide Web Consortium)

The international standards organization founded by Tim Berners-Lee in 1994 to develop web standards. The W3C published HTML 3.2, HTML 4.01, XHTML 1.0, and HTML5 as formal Recommendations.

WHATWG (Web Hypertext Application Technology Working Group)

A community founded by engineers from Apple, Mozilla, and Opera in 2004 to continue practical HTML development when W3C moved away from HTML toward XHTML 2.0. WHATWG maintains the HTML Living Standard.

HTML Living Standard

The continuously updated HTML specification maintained by WHATWG. Rather than numbered versions, the specification evolves as browsers implement new features and tests confirm interoperability.

Semantic HTML

Using HTML elements that convey meaning about the content they contain, not just its appearance. Examples include <article>, <nav>, and <aside> introduced in HTML5.
