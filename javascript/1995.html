1995 – The Birth of JavaScript

In May 1995, Brendan Eich created JavaScript at Netscape Communications in approximately ten days. The World Wide Web at that time consisted almost entirely of static HTML documents. Interaction required server requests, and user input triggered full page reloads. JavaScript introduced client-side execution — code running directly in the browser without contacting a server.

Originally named Mocha, then LiveScript, the language was renamed JavaScript shortly before release to align with the marketing momentum of Sun Microsystems' Java. Despite the shared name, JavaScript was never a subset of Java. Its design combined influences from Scheme (functional patterns), Self (prototype-based inheritance), and C-style syntax.

There was no formal standard in 1995. Each browser implemented its own behavior. The language was designed quickly, without anticipating the scale of applications it would eventually support.

Technical Characteristics of 1995 JavaScript
  • Prototype-based object model (no formal class syntax)
  • Dynamic, loosely typed variables
  • First-class functions
  • Everything placed in the global scope by default
  • Direct DOM manipulation via document.write() and inline event handlers
  • No modules, no error handling, no standard library

Why This Matters

Despite its hasty creation, JavaScript's placement inside Netscape Navigator 2.0 made it the first programming language millions of people encountered. Its presence in the browser — not its elegance — determined its future. By the time anyone had a better idea, JavaScript was already everywhere.
