Glossary

Key terms used throughout the JavaScript history archive.

ECMAScript

The formal name for the JavaScript language specification, maintained by Ecma International under the designation ECMA-262. "JavaScript" is a trademark of Oracle Corporation; the standardized language is officially called ECMAScript.

TC39

Technical Committee 39 of Ecma International. Responsible for evolving the ECMAScript specification. Members include representatives from Google, Mozilla, Apple, Microsoft, Meta, and others.

Prototype-based inheritance

An object-oriented model in which objects inherit directly from other objects rather than from class definitions. JavaScript uses prototype chains: every object has an internal link to a prototype object, and property lookups traverse this chain.

Transpiler

A tool that converts code written in one version of a language into an equivalent version supported by older runtimes. Babel is the most widely used JavaScript transpiler, allowing ES2015+ syntax to compile to ES5 for older browsers.

Strict mode

An opt-in mode introduced in ES5, activated by placing "use strict"; at the top of a script or function. Strict mode turns silent errors into exceptions and forbids certain unsafe syntax.

DOM (Document Object Model)

An API provided by the browser that represents an HTML document as a tree of objects. JavaScript uses the DOM to read and modify page content, structure, and styling at runtime.

Promise

An object introduced in ES2015 representing the eventual result of an asynchronous operation. A Promise is pending, fulfilled (with a value), or rejected (with a reason).

async/await

Syntax introduced in ES2017 that allows asynchronous code to be written like synchronous code. An async function always returns a Promise; await pauses execution inside it until a Promise settles.

JScript

Microsoft's implementation of JavaScript, introduced in Internet Explorer 3.0 in 1996. JScript included proprietary extensions and differed from Netscape's implementation in ways that created significant compatibility problems.

ECMA-262

The official designation of the ECMAScript Language Specification document. The number 262 is the Ecma project number assigned when JavaScript was first submitted for standardization in 1996.

Polyfill

JavaScript code that implements a modern API in environments that do not natively support it. Unlike a transpiler (which converts syntax), a polyfill adds missing runtime functionality.

Event loop

The mechanism that allows JavaScript — a single-threaded language — to handle asynchronous operations. The event loop continuously checks a message queue and executes callbacks when their corresponding events have completed.
