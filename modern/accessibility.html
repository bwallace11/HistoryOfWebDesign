Accessibility in Web Design

Accessibility means designing and building websites that people with different abilities can use with dignity, independence, and equal access.

It is not a bonus feature: it is a civil rights issue because digital services now shape education, healthcare, jobs, government services, and daily life.

Key milestones

  • WAI (Web Accessibility Initiative) launched by W3C in 1997 to coordinate global accessibility standards and education.
  • WCAG 1.0 (1999) established foundational accessibility guidance for content and design.
  • WCAG 2.0 (2008) introduced technology-neutral success criteria organized by POUR principles.
  • WCAG 2.1 (2018) expanded coverage for mobile interaction, low vision needs, and cognitive accessibility.
  • ADA relevance: in many jurisdictions, public-facing websites are interpreted as places of public accommodation and subject to equal-access expectations.

POUR principles

Perceivable

Information must be presented in ways users can detect, including alternatives for images, captions for media, and adaptable content.

Operable

Interfaces must work with keyboard and assistive technologies, with clear focus states and enough time to complete tasks.

Understandable

Navigation and language should be predictable and clear, with helpful error messages and consistent patterns.

Robust

Content should use standards-compliant markup so it stays compatible with current and future browsers and assistive tools.

Real examples in practice

  • Alt text: meaningful descriptions for informative images.
  • Semantic HTML: headings, landmarks, lists, and forms with proper labels.
  • Keyboard navigation: no mouse-only controls; visible focus.
  • ARIA: used to enhance semantics only when native elements cannot express behavior.
  • Color contrast: readable text and UI states for low-vision users.
