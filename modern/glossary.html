Glossary

Short definitions for core web design and accessibility terms.

HTML
The structure layer of the web that defines content and meaning.
CSS
The styling layer that controls layout, color, spacing, and visual presentation.
JavaScript
The behavior layer that enables interactivity and dynamic updates.
Accessibility
Designing and building the web so people with disabilities can use it effectively.
WCAG
Web Content Accessibility Guidelines, a widely used accessibility standard.
Screen Reader
Assistive software that reads digital content using speech or braille output.
Semantic HTML
Using meaningful elements like headings, landmarks, and lists for clear structure.
Responsive Design
Layouts that adapt to different screen sizes and device contexts.
Progressive Enhancement
Building a reliable baseline experience, then enhancing for capable devices.
ARIA
Accessibility attributes used when native HTML semantics are not enough.
DOM
The in-memory document model that scripts use to read and change page content.
UX / UI
UX is the overall user experience; UI is the interface users directly interact with.
Dark Patterns
Interface choices that manipulate users into actions they might not freely choose.

The early web often highlighted technical possibility and rapid innovation. These current-era pages exist to foreground the human impact of design: accessibility makes the web usable for everyone, the future section tracks where design is heading, and the glossary/resources pages support transparent, practical learning.
