Bibliography / Resources

Selected references for web standards, accessibility practice, and modern design ethics.

  1. W3C WCAG Overview — Primary standards entry point for understanding accessibility requirements and versions.
  2. W3C Web Accessibility Initiative (WAI) — Central hub for guidance, tutorials, and accessibility planning frameworks.
  3. WebAIM — Practical articles, checklists, and testing resources used widely by developers and educators.
  4. MDN Web Docs — Reliable technical reference for semantic HTML, ARIA, CSS, and JavaScript best practices.
  5. A List Apart — Influential writing archive on web standards, content strategy, and responsible UX.
  6. WAI-ARIA Specification — Defines accessible semantics for advanced widgets and dynamic interfaces.
  7. Adrian Roselli Blog — Deep, implementation-focused critiques and guidance for real production accessibility issues.
  8. Scott O’Hara Articles — Detailed analyses of ARIA and interaction semantics for modern components.
  9. Hidde de Vries Blog — Clear writing on inclusive components, browser behavior, and practical front-end choices.
  10. The A11Y Project — Community-curated accessibility checklists, patterns, and beginner-friendly explanations.

The early web often highlighted technical possibility and rapid innovation. These current-era pages exist to foreground the human impact of design: accessibility makes the web usable for everyone, the future section tracks where design is heading, and the glossary/resources pages support transparent, practical learning.
